container

임시의 작은 보라색 공간에 웹 페이지의 전체라고 생각하면 곧 이 전체에 해당하는 경계에 감싸고 있는 임의의 선을 정해 놓고 이 선을 "큰 상자" 라는 뜻의 container로 지정했다.

이렇게 임의의 선 또는 박스들을 구조물로써 배치 하는것이 html의 역할이라고 본다.

곧, 시맨틱태그의 이해와 구조 설정을 잘 해야 나중에 User Interfcae를 잘 고려한 웹 사이트를 만들 수 있는 가능성이 높다. 물론 작업자의 의도에 맞게 꼭 container일 필요는 없다.

이 작업은 3번째와 4번째 항목과 깊은 연관이 있다 html을 배우고 나면 css를 배우게 될 것이다.

좀 많이 건너뛰어서, 왜 html이 중요한가에 대해 내 생각을 적어보려고 한다.

html, css 더 나아가 Javacript를 배우고 나면 프레임 워크 라는 것을 배우게 될텐데 Vue, Angular.Js, React 등이 있다. 그 중 나는 REACT를 예시로 생각을 적어보자면
React에서는 JSX 문법을 사용하게 된다. JSX 문법이란 Javascript 문법의 확장한 문법으로 Javacript + XML 형태인 확장자 이다.

XML이란?

  • XML(eXtensible Markup Language) 은 HTML과 같은 마크업 언어이지만, HTML은 데이터를 표현하는 마크업 언어라면, XML은 데이터를 기술하는 언어이다.
  • 예를 들어 HTML에서는 h1, p 같은 이미 만들어진 태그를 사용해야 하지만, XML은 정의된 부분을 제외하고는 사용자가 태그를 임의로 만들 수 있다.
  • 엄밀히 말하자면 XML은 마크업 언어라기 보다 마크업 언어를 정의하기 위한 언어라고 할 수있다.

리액트를 배우게 될 시점은 크게 두가지 조건이 충족됐을 때 라고 생각한다.

  1. Javacript에 친숙해졌을 때
  2. 웹 구성요소를 이해하고 구조화에 어려움이 없을 때

리액트를 배울 때 가장 놀랬던 점은 jsx문법에서의 대부분은 html + js 문법으로 이루어져 있으며 좀 더 간편한 사용을 위해 여러가지 기능과 함수가 추가된 느낌이었다. 이 말은 곧 js와 친해지고 html을 쓸 수 있으면 리액트는 더 재밌게 접근할 수 있다고 생각한다. 하지만 반대로 웹 구조를 체계적으로 생각하지 못하면 리액트로 넘어가도 힘들 수 있다고 생각한다.

CSS와 TailwindCSS

CSS

CSS 는 Cascading Stlye Sheet의 줄임말로 cascade의 사전적 의미는 작은 폭포, cascading은 위에서 아래로 흐르는, 상속, 종속을 의미한다. 그러나 지금 우리가 여기서 언급하는 캐스캐이딩은 이러한 사전적 의미와는 다소 다르다. 쉽게 말하자면 우선순위가 높은 속성 순서대로 스타일을 입힌다 라고 해석할 수 있다.

내가 경험했던 css 사용법은 크게 3가지로 설명할 수 있다.

  1. html 내 Style 태그를 이용한 방법
  2. html 태그 안에 간단한 인라인 요소로 사용한 방법
  3. css 파일을 import해서 사용한 방법
각 방법마다 장단점이 존재하는데, 이 문서에서는 3번을 소개하려고 한다.
그 이유는 css를 가장 직관적으로 관리할 수 있다고 생각해서이다. 우선 import해서 사용하는 방법도 여러가지가 존재하는데 내가 사용한 방법은 css module 인데 사용한 이유는 크게 2가지 이다.
  1. 클래스명 충돌 방지
  2. 컴포넌트 단위의 스타일을 지정할때 간편함
물론 더 나아가다 보면 뒤에 서술할 TailwindCSS 라는 것이 등장하는데,
이 TailwindCSS는 위에 2가지 중 첫 번째 이유를 커버하는 프레임워크이다. 각설하고 css module을 간단히 보자면 css 파일을 import해서 클래스명에 따라 지정해주면 된다.
